## Finding: A11Y-009 — Bewegung, Animation und Auto-Updates kontrollierbar

| Feld | Wert |
|---|---|
| **Severity** | high (Basis medium, eCH-0059-Eskalation) |
| **Status** | closed (PR #205; Re-Audit 2026-07-16 live bestätigt: reduced-motion-CSS deployt) |
| **Lösung** | `maschinerie-zuerich` |
| **Check-Reference** | `A11Y-009` |
| **WCAG-Ref** | 2.2.2 (A), 2.3.3 (AAA) |
| **Audit-Datum** | 2026-07-15 |
| **Auditor** | Claude (ux-audit v0.1.0) |
| **Journey / Ort** | ganze App, mehrere animierte Views |

### Observed Behavior

`prefers-reduced-motion` kommt im gesamten Repository **kein einziges Mal** vor:

```
grep -r "prefers-reduced-motion" stadt-zuerich-next/  -> 0 Treffer
```

Gleichzeitig enthält die App mehrere animierte/bewegte Komponenten (grep-Treffer für `animate`/Transitions/Graph-Layout):
`GraphView.tsx`, `TreemapView.tsx`, `ProzessFlow.tsx`, `Onboarding.tsx` (react-joyride Beacon mit Pulsier-Animation), `BudgetSimulator.tsx`, Theme-Übergänge. Kein globaler Media-Query-Guard, der diese Bewegung bei gesetzter System-Präferenz reduziert.

### Expected Behavior

Nutzer mit `prefers-reduced-motion: reduce` (vestibuläre Beschwerden) sollen grossflächige/kontinuierliche Animationen abgeschaltet oder stark reduziert bekommen. WCAG 2.2.2 verlangt zudem Pause/Stopp für automatisch bewegte Inhalte > 5s (betrifft v.a. den kontinuierlich pulsierenden Onboarding-Beacon).

### Evidence

- Repo-Grep (ganzes Projekt): `prefers-reduced-motion` = 0 Treffer
- Laufzeit-Check: `[...document.styleSheets]` enthält keine `prefers-reduced-motion`-Regel (`reducedMotionCSS: false`)
- Animierte Komponenten: `components/GraphView.tsx`, `components/TreemapView.tsx`, `components/Onboarding.tsx` (`react-joyride__beacon`)

### Risk Description

Für Menschen mit vestibulären Störungen können der pulsierende Beacon und Layout-/Graph-Animationen Schwindel/Übelkeit auslösen. Bei `compliance: eCH-0059` ist Barrierefreiheit gesetzliche Pflicht der öffentlichen Hand — ein fehlender Reduced-Motion-Pfad ist ein dokumentationspflichtiger Mangel.

### Remediation

Globale CSS-Weiche ergänzen (einmalig, greift für alle Komponenten):

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

Für den Onboarding-Beacon zusätzlich: bei gesetzter Präferenz statisches Icon statt Pulsanimation. Für d3/cytoscape-Layouts (`GraphView`, `TreemapView`): Animationsdauer der Layout-Transition an die Präferenz koppeln (`prefersReducedMotion ? 0 : 400`).

**S** — eine globale CSS-Regel deckt den Grossteil ab; Beacon/Graph-Feinschliff je S.
